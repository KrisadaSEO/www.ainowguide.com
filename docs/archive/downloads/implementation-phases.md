# Krisada.com Rebuild Implementation Phases

## Phase 1 ... Foundation

Build the smallest durable system.

### Deliverables

- repo structure
- bootstrap and router
- shared layout shell
- design tokens
- homepage template
- page and article template
- category template

### Outcome

The site renders with stable structure.

## Phase 2 ... Content library

### Deliverables

- category schema
- article schema
- nested category support
- breadcrumbs
- master library page
- category and subcategory landing pages
- related content logic

### Outcome

The library becomes the backbone of the site.

## Phase 3 ... Sidebar system

### Deliverables

- three highlight block styles
- sidebar profile resolver
- Thai/local/legal blocks
- template assignment rules

### Outcome

The site gets guided relevance and stronger conversions.

## Phase 4 ... Redirects and migration support

### Deliverables

- redirect schema
- import script
- validation script
- early request redirect matching

### Outcome

Legacy traffic and old URLs are preserved cleanly.

## Phase 5 ... Federation integration

### Deliverables

- site federation config
- `/ai/` output files
- federation build script
- metadata injection support

### Outcome

Krisada.com becomes AI machine-readable in a structured, maintainable way.

## Phase 6 ... GitHub automation

### Deliverables

- deploy workflow
- validation workflow
- sitemap build flow
- post-deploy build steps

### Outcome

Publishing becomes safer and faster.

## Phase 7 ... Polish and cloneability

### Deliverables

- cleanup of domain-specific strings
- install variables or starter config model
- starter prompts for new domains
- optional reusable install package

### Outcome

The system starts becoming the seed of a multi-domain installer.
