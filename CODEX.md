# Codex Session Rules

Read this file before doing anything in this repo each session.

## Git workflow

- NEVER create branches.
- Always work directly in `main`.
- Commit directly to `main`.
- Push directly to `main`.
- This is a private repo with no parallel development.
- There is no need for feature branches.
- Do not open pull requests unless explicitly requested.
- If the user says "push", that means push to `main`.

## Working style

- Keep the workflow simple and direct.
- Follow `AGENTS.md` before making changes.
- Respect generated-file rules and regenerate outputs when required.
